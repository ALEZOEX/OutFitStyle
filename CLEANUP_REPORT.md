# Отчет о чистке проекта OutfitStyle

## Цель
Наведение порядка в проекте: удаление временных, устаревших и дублирующих файлов, унификация структуры.

## Удаленные файлы

### Миграции
- Удалены старые миграции (001_, 002_, 003_ и т.д.)
- Оставлена только одна основная миграция: `0001_v2_init_schema.sql`

### Временные/технические файлы
- Удален исполняемый файл `server.exe`
- Удалены временные файлы конфигурации и документов
- Удалены тестовые скрипты с суффиксом `_test`
- Удалены временные конфигурационные файлы

### Документация
- Удален дублирующийся README.md из папки server
- Оставлен только основной README в корне
- Файл database/README.md перемещен в docs/database_schema.md

### Исходный код
- Удалены старые версии файлов (с суффиксами v1, v2, original, legacy)
- Удалены временные/черновые реализации
- Удалены временные файлы ML-модели
- Удалены временные скрипты

## Оставлены только актуальные файлы

### Сервер (Go)
- `/cmd/` - точка входа
- `/internal/` - внутренние пакеты
  - `/api/` - HTTP слой
  - `/core/` - бизнес-логика
  - `/infrastructure/` - инфраструктурные компоненты
- `/migrations/` - миграции БД
- `/database/` - файл инициализации

### ML-сервис (Python)
- `/api/` - FastAPI приложение
- `/model/` - ML компоненты
- `/contracts/` - контракты API
- `/data/` - данные для обучения
- `/train/` - скрипты обучения

### Клиент (Flutter)
- `/lib/` - исходный код
  - `/screens/` - экраны приложения
  - `/services/` - сервисы
  - `/models/` - доменные модели

### Документация
- `/docs/` - основная документация
- `/README.md` - основное описание проекта
- `/CONTRIBUTING.md` - руководство по участию
- `/LICENSE` - лицензия

## Структура теперь соответствует production-ready стандартам

1. **Четкая архитектура**: planner → retrieval → ranking
2. **Единая точка инициализации БД**: через 0001_v2_init_schema.sql
3. **Чистый код**: только актуальные реализации сервисов
4. **Правильные контракты**: Go ↔ ML через well-defined API
5. **Безопасность**: все проверки авторизации и аутентификации
6. **Наблюдаемость**: метрики, логи, healthchecks
